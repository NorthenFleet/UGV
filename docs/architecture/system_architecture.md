# 系统架构总览

本系统由三大端与一个桥接层构成：
- 仿真与训练端（Mac/Linux）：基于 PyBullet 的六足机器人环境，采样并训练 RL 策略
- 运行端（Jetson/Raspberry Pi）：加载训练好的策略，进行实时状态估计与控制下发
- 控制端（STM32）：舵机/PWM 驱动与底层安全保护，状态反馈上报
- ROS2 桥接层（可选）：统一话题与工具链，将仿真/实机的数据与指令桥接

核心目标：在仿真中学习稳定高效的步态与导航策略，并在运行端以安全约束实现近实时时间控制，最终驱动实体六足机器人。

关键特性：
- 可插拔仿真后端（PyBullet 为默认，MuJoCo 可选）
- 标准化观察/动作/奖励接口，支持 curriculum 学习
- Jetson/RPi 端包含安全限幅与急停，确保实机可靠
- STM32 固件独立可运行，具备通信异常时的保形策略
